WIN 64bit 체제에서 작동하는 **DirectX 11 기반 게임 엔진**을 개발하기 위한 요구사항을 정리했다.

## 주요 기능

1. **PIE (Play In Editor)** - 에디터 내 플레이 테스트
2. **월드 아웃라이너** - 액터 관리
3. **액터와 액터 배치** - 오브젝트 편집
4. **레벨 관리** - 스테이지 관리
5. **패키징 및 빌드** - exe 파일 배포

---

## Use Case

### 주요 액터(Actors)

| 액터 | 설명 |
|------|------|
| **사용자(User)** | 에디터를 사용하여 레벨을 생성하고 테스트 |
| **게임 엔진(Game Engine)** | DirectX 11 기반으로 사용자 요청을 처리 |

### Use Case 목록

#### 1. PIE (Play In Editor) 실행
- 사용자가 현재 편집 중인 레벨을 에디터 내에서 실행하여 테스트

#### 2. 월드 아웃라이너 관리
- 씬 내 존재하는 액터를 목록화하여 관리
- 액터를 선택, 삭제, 정렬 가능

#### 3. 액터 배치 및 편집
- 새로운 액터를 추가하고, 위치, 크기, 회전 등을 조정

#### 4. 레벨 관리(스테이지 관리)
- 새로운 레벨을 생성, 저장, 불러오기 기능 제공

#### 5. 패키징 및 빌드
- 최종적으로 `.exe` 파일을 생성하여 외부 배포 가능하도록 함

## 기능 상세 정보

### 액터(Actor)란?

액터는 **게임 내에서 오브젝트(객체) 역할을 하는 요소**다. 게임 환경에서 배치되고 조작 가능한 모든 요소를 의미한다.

> 예: 플레이어, 장애물, 배경 오브젝트, 조명, 카메라 등

### 액터의 주요 구성 요소

| 컴포넌트 | 필수 | 설명 |
|----------|------|------|
| **트랜스폼(Transform)** | ✅ | 위치(Position), 회전(Rotation), 크기(Scale) 정보 |
| **메시(Mesh)** | | 3D 모델 데이터 (큐브, 스피어, 캐릭터 모델 등) |
| **머티리얼(Material)** | | 액터의 외형을 결정하는 재질 |
| **물리(Physics)** | | 충돌(Collision), 중력(Gravity) 등 물리 연산 |
| **스크립트(Script)** | | 액터의 행동을 결정하는 로직 |

---

### 레벨(Level)이란?

레벨은 **게임 플레이가 이루어지는 공간**을 의미하며, 여러 액터들이 배치되고 게임 규칙이 적용되는 단위다.

### 레벨의 주요 구성 요소

1. **월드 설정**
   - 중력, 조명, 스카이박스(하늘 표현), 바닥 설정 등

2. **지형(Terrain) 또는 플랫폼**
   - 플레이어가 이동할 수 있는 땅, 구조물 등

3. **액터(Actors)**
   - 장애물, 플레이어, 오브젝트, 카메라 등 배치

4. **이벤트 트리거(Event Triggers)**
   - 특정 지점 도달 시 이벤트 발생 (애니메이션, 소리, UI 변화 등)

5. **게임 로직 및 규칙**
   - 스폰 포인트, 목표 지점 등 게임 진행 방식 정의
